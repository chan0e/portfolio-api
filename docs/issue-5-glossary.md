## 식별자/관계 용어
| 용어 | 분류 | 정의 |
|---|---|---|
| `table_name_id` | 명명 규칙 | 논리 모델에서 PK 이름을 `테이블명_id` 형태로 통일하는 규칙 |
| `PK` | 키 | 각 테이블 레코드를 유일하게 식별하는 기본키(Primary Key) |
| `FK` | 키 | 부모 테이블 키를 참조하는 외래키(Foreign Key) |
| `UNIQUE` | 제약 | 지정 컬럼(또는 컬럼 조합)의 중복을 금지하는 제약 |
| `1:1` | 카디널리티 | 한 부모 레코드가 정확히 한 자식 레코드와 대응되는 관계 |
| `1:N` | 카디널리티 | 한 부모 레코드가 여러 자식 레코드와 대응되는 관계 |
| `ON DELETE CASCADE` | 참조 정책 | 부모 삭제 시 연결된 자식 레코드를 함께 삭제하는 FK 옵션 |
| `Crow's Foot (IE)` | 표기법 | ERD에서 관계 수량을 `||`, `o{` 등으로 표현하는 표기 체계 |
| `||` | IE 기호 | 정확히 1(exactly one) 관계를 의미 |
| `o{` | IE 기호 | 0개 이상(zero or many) 관계를 의미 |

## 컬럼/모델링 용어
| 용어 | 분류 | 정의 |
|---|---|---|
| `display_order` | 정렬 컬럼 | 리스트 응답 순서를 보장하기 위한 정렬 기준 컬럼 |
| `portfolio_key` | 비즈니스 키 | 포트폴리오 루트를 식별하는 유니크 키(슬러그 형식) |
| `source` | 메타 컬럼 | 동기화 데이터의 출처 정보(예: notion) |
| `synced_at` | 메타 컬럼 | 외부 소스와 동기화된 시각 |
| `created_at` | 감사 컬럼 | 레코드 생성 시각 |
| `updated_at` | 감사 컬럼 | 레코드 최종 수정 시각 |
| `start_date` | 기간 컬럼 | 경력 시작일 |
| `end_date` | 기간 컬럼 | 경력 종료일(재직 중이면 `NULL`) |
| `is_current` | 상태 컬럼 | 현재 재직 상태 여부 |
| `slug format` | 검증 규칙 | 소문자/숫자/`_`/`-` 허용 패턴으로 문자열 형식을 제한하는 규칙 |
| `derived period` | 매핑 개념 | API의 `period` 문자열을 DB의 날짜 컬럼 조합으로 파생해 생성하는 방식 |

## 제약/검증 용어
| 용어 | 분류 | 정의 |
|---|---|---|
| `NOT NULL` | 제약 | 컬럼에 `NULL` 저장을 금지하는 제약 |
| `CHECK` | 제약 | 컬럼 값이 특정 조건을 만족해야만 저장되도록 강제하는 제약 |
| `btrim(field) <> ''` | 검증 규칙 | 공백만 있는 문자열 저장을 금지하기 위한 체크 패턴 |
| `octet_length(...) > 0` | 검증 규칙 | `BYTEA` 암호문 컬럼이 빈 값이 아니도록 강제하는 체크 |
| `date range check` | 검증 규칙 | `start_date/end_date/is_current` 조합의 기간 정합성을 보장하는 체크 |
| `at least one URL` | 검증 규칙 | `project_link`에서 `github_url` 또는 `demo_url` 중 최소 1개 필수 조건 |
| `composite unique` | 제약 패턴 | 두 개 이상 컬럼 조합으로 중복을 제어하는 유니크 제약 |

## 보안/암호화 용어
| 용어 | 분류 | 정의 |
|---|---|---|
| `PII` | 보안 | 개인 식별 가능 정보(개인정보) |
| `*_enc` | 암호화 컬럼 | 평문 대신 저장되는 암호문 페이로드 |
| `*_iv` | 암호화 컬럼 | 암호화 초기화 벡터(IV) 저장 컬럼 |
| `*_tag` | 암호화 컬럼 | 인증 태그(Tag) 저장 컬럼 |
| `*_key_version` | 키 관리 | 암호화 키 버전을 추적하는 컬럼 |
| `email_hash` | 검색 보조 | 이메일 평문 검색 없이 동일성 조회를 지원하는 해시 컬럼 |
| `AES-256-GCM` | 암호 알고리즘 | 인증 암호화(AEAD) 방식으로 IV/Tag와 함께 사용되는 대칭키 알고리즘 |
| `KMS/Vault` | 비밀 관리 | 암호화 키를 DB 외부에서 관리하는 키/비밀 관리 시스템 |

## 운영/배포 용어
| 용어 | 분류 | 정의 |
|---|---|---|
| `MVP` | 개발 단계 | 핵심 기능 중심으로 최소 구현을 우선하는 단계 |
| `ddl-auto=update` | 개발 설정 | JPA가 엔티티 기반으로 스키마를 자동 갱신하는 설정 |
| `마이그레이션 기반 관리` | 운영 전략 | 스키마 변경을 버전 스크립트로 명시적으로 관리하는 방식 |
| `트리거` | DB 객체 | 특정 이벤트(예: UPDATE) 시 자동 실행되는 DB 로직 |
| `set_updated_at()` | 트리거 함수 | UPDATE 시 `updated_at`을 현재 시각으로 갱신하는 공통 함수 |
| `EXPLAIN (ANALYZE, BUFFERS)` | 성능 진단 | 실제 실행 계획/버퍼 사용량을 확인해 인덱스 필요성을 판단하는 명령 |
