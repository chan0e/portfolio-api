# DB 설계 리뷰 체크리스트

## 계약 정합성
- API 필수 필드가 DB의 `NOT NULL` 또는 동등한 제약으로 보장되는지 확인하라.
- 선택 필드가 `NULL` 처리와 직렬화 규칙에 맞게 설계되었는지 확인하라.
- 배열/목록 정렬 규칙이 DB 정렬 기준으로 재현되는지 확인하라.

## 모델링
- 카디널리티(`1:1`, `1:N`, optional)가 비즈니스 규칙과 일치하는지 확인하라.
- 부모-자식 삭제 정책(`ON DELETE`)이 명시되어 있는지 확인하라.
- 키/타임스탬프/정렬 컬럼 네이밍 규칙이 일관적인지 확인하라.

## 제약조건
- 핵심 불변식이 DB 제약조건으로 강제되는지 확인하라.
- 유니크 제약에 범위 키(예: portfolio_id)가 빠지지 않았는지 확인하라.
- `CHECK` 제약으로 비정상 값이 차단되는지 확인하라.

## 인덱스
- 인덱스가 실제 쿼리 패턴 기반인지 확인하라.
- `PRIMARY KEY`/`UNIQUE`와 중복되는 인덱스가 없는지 확인하라.
- 신규 인덱스의 쓰기 비용과 유지보수 비용이 문서화되었는지 확인하라.

## 마이그레이션 안정성
- 롤링 배포 기준 하위 호환성이 확보되는지 확인하라.
- 롤백/백필 절차가 문서화되었는지 확인하라.
- 타입 변경/제약 강화 시 데이터 변환 계획이 있는지 확인하라.

## 보안
- 민감정보가 암호화 저장 및 키 버전 관리 정책을 따르는지 확인하라.
- 검색이 필요한 민감정보는 해시 보조 컬럼 정책이 있는지 확인하라.
- 비밀값/키가 소스 코드에 하드코딩되지 않았는지 확인하라.

## 검증
- 대표 쿼리에 대해 실행 계획 검토(`EXPLAIN ANALYZE`)가 가능한지 확인하라.
- API DTO와 DB 모델 간 매핑 통합 테스트가 있는지 확인하라.
- null/빈배열/중복 순서/고아 레코드 등 엣지 케이스를 검증했는지 확인하라.
